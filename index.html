3 Concepts

3 Main concepts

    Store Hold the state of an application
    ActionDesribe what will happen when state will change
    ReducerIt Ties the store and Action together

Three Main Principle

1st Principle

    The state of whole application is store in a single object which will be manage by the Redux store
2nd Principle
    The only way to change state is by emiting action an object descript what happens
    To update state you need to let redux know about with an action
    Not allow to update state object directly
3rd Principle
    To update state write pure Reducer
    To specify how the state will transfer by action write your Reducer
    Reducer (preState + Action)=> newState

Action

    An action is an Object with type properties and Action Creator is an function that return the function
    Type property indicate the action to be perform

Reducer

    It specify how app change or responce to action sent to redux store
    Function that accept state and argument and return the next state of application
    (prevActin,State)=>newState
